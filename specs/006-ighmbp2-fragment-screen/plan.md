# Implementation Plan: IGHMBP2 Fragment-Based Virtual Screening

**Branch**: `006-ighmbp2-fragment-screen` | **Date**: 2026-05-29 | **Spec**: [spec.md](spec.md)

## Summary

Add a `pipeline fragment` command that runs a 6-step fragment-based virtual screen against the IGHMBP2 AlphaFold2 structure, producing `compounds_filtered.csv` in VRK1 schema ready for `pipeline dock`. Steps: pocket identification (fpocket) → fragment library download (ZINC) → batch docking (Vina exhaustiveness=4) → Tanimoto clustering (Butina) → fragment growing (BRICS + SMARTS) → ADMET screening (existing gate). All computation runs locally on M1 Max; full pipeline completes in ~8–14 hours.

## Technical Context

**Language/Version**: Python 3.11 (existing project)

**Primary Dependencies (new)**:
- `sa-score` (pip) — synthetic accessibility scoring via `sascorer` module
- `fpocket` (already installed as CLI tool, invoked via subprocess)

**Dependencies (existing, reused)**:
- `rdkit` — BRICS, Butina clustering, ECFP4 fingerprints, SMARTS reactions, Murcko scaffolds, Ro3 filtering
- `vina`, `meeko` — fragment docking (reuse `_prepare_receptor`, `_prepare_ligand`, `_run_vina` from dock stage)
- `admet-ai` — ADMET screening (reuse `run_admet_gate` from validate stage)
- `httpx`, `tenacity` — ZINC library download
- `biopython` — PDB parsing for pocket centroid extraction
- `pandas`, `rich` — data handling and console output

**Storage**: `data/results/IGHMBP2/` and `data/cache/shared/fragment_library/` (shared across targets)

**Testing**: pytest (existing). Unit tests for each step module.

**Target Platform**: macOS Apple M1 Max 64 GB. All computation local.

**Performance Goals**:
- pocket step: < 60 s
- library step: < 10 min
- dock step: < 12 h (8,000 fragments × ~4 s/fragment at exhaustiveness=4)
- cluster step: < 30 s
- grow step: < 15 min
- admet step: < 5 min

**Constraints**: No cloud, no paid APIs. Interruption-safe (cache per fragment).

**Project Type**: CLI pipeline extension (existing `pipeline/stages/` pattern)

## Key Implementation Decisions

### Step 1 — Pocket (pocket.py)
- Subprocess: `fpocket -f {af2_pdb_path}` → parse `{stem}_out/{stem}_info.txt`
- Parse whitespace-delimited info file with pandas; columns: Score, Druggability_Score, Volume
- Select top pocket where Volume > 200 Å³, sorted by Druggability_Score
- Extract centroid from `{stem}_out/pockets/pocket{N}_atm.pdb` (mean of all heavy atom coords)
- pLDDT filter: load AlphaFold JSON (already cached), compute mean pLDDT of residues within 8 Å of centroid; warn if < 70
- Write `pocket_analysis.json` with centroid, volume, score, box_size=20.0

### Step 2 — Library (library.py)
- Download from ZINC22 tranches at `https://files.docking.org/2D/AA/AAAA.smi` etc.
- For simplicity: use a pre-selected set of 10 ZINC fragment tranche files covering fragment-like MW range
- Parse SMILES line by line; apply Ro3 filter via RDKit descriptors
- Murcko scaffold deduplication: `Chem.Scaffolds.MurckoScaffold.GetScaffoldForMol`
- Random sample to `--library-size` (default 8,000)
- Fallback: read `data/cache/shared/fragment_library/fragments_fallback.smi` (committed, 500 compounds)

### Step 3 — Screen (screen.py)
- Reuse `_prepare_receptor`, `_prepare_ligand`, `_run_vina`, `_define_box` from `pipeline/stages/dock/dock.py`
- Box centre from pocket centroid; box size 20 Å (larger than kinase ATP pocket)
- Exhaustiveness=4 (configurable via `--exhaustiveness`)
- Cache key per fragment: `fragment_dock_{fragment_id}` in `data/cache/IGHMBP2/`
- Failures (meeko parametrisation error, vina crash): log and skip; count in report
- Top-N selection: sort all results by affinity, take top `--top-n` (default 50)

### Step 4 — Cluster (cluster.py)
- Morgan ECFP4, radius=2, nBits=1024
- Butina clustering, cutoff=0.4 (60% similarity)
- Representative = fragment with best affinity in each cluster
- Write `fragment_clusters.csv`

### Step 5 — Grow (grow.py)
- Install `sa-score` pip package; import `sascorer`
- BRICS approach: `BRICS.BRICSBuild([representative_mol, building_block])` with a curated set of 20 drug-like building blocks (committed as `building_blocks.smi`)
- SMARTS approach: apply 15 transformations (amide, sulfonamide, N-alkyl, aryl-F, CF3, ring closure)
- Filter: MW 300–450, Ro5 (0 violations), RotB ≤ 8, SA score < 4
- Keep top 3 by estimated size-adjusted score per representative
- Write `grown_candidates.csv`

### Step 6 — Output (output.py)
- Reuse `run_admet_gate` from `pipeline/stages/validate/gates/admet.py`
- Apply BBB threshold 0.3 (relaxed; documented in report)
- Write `compounds_filtered.csv` in exact VRK1 schema
- Write `fragment_screen_report.md` with: pocket summary, top-10 fragments table, cluster summary, grown candidates table, ADMET results, next-steps section

### CLI (cli.py)
- `pipeline fragment` command following existing pattern
- Options: `--target`, `--step`, `--force`, `--top-n`, `--exhaustiveness`, `--library-size`, `--data-dir`
- Add `"Bash(pipeline fragment *)"` to `.claude/settings.local.json`

## Complexity Tracking

| Design choice | Why needed | Simpler alternative rejected |
|---|---|---|
| 6-step modular pipeline | Each step has different runtime (seconds to hours); cache per step enables interruption recovery | Monolithic script would restart from zero on any failure |
| Reuse existing dock stage | PDBQT prep, receptor caching, Vina API already battle-tested | Re-implementing would duplicate 300+ lines and reintroduce PDBQT column bugs |
| BRICS + SMARTS dual approach | BRICS is combinatorial (faster, more conservative); SMARTS reactions add medicinal chemistry diversity | BRICS alone produces too-similar analogs; SMARTS alone is harder to validate |
| Bundled fallback library | ZINC download may fail or be slow; 500 fragments lets the pipeline run end-to-end for testing | No fallback would block testing in offline/CI environments |
| Shared fragment library cache | Library is target-agnostic; re-downloading for each target wastes time | Target-specific library would prevent reuse if VCP is later screened |
