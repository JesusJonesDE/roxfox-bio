## Summary

An idempotent Python CLI pipeline that fetches bioactivity, structural, genetic, and competitive landscape data from public APIs (ChEMBL, Open Targets, PDB, AlphaFold, ClinicalTrials.gov) for three pre-configured drug targets (VRK1, IGHMBP2, VCP). Applies cheminformatics analysis (Lipinski filtering, Murcko scaffold clustering, selectivity profiling) and generates per-target research dossiers. All API responses are cached to disk; re-runs skip stages whose data is still fresh.

---

## Technical Context

**Language/Version**: Python 3.11+

**Primary Dependencies**:
- `typer` + `rich` — CLI framework and progress display
- `httpx` + `tenacity` — HTTP client with retry/backoff
- `rdkit` (conda-forge) — cheminformatics (Lipinski, Murcko scaffolds)
- `pandas` — CSV output
- `jinja2` — markdown dossier templating

**Storage**: Local filesystem only — JSON cache files + CSV results + markdown dossiers

**Testing**: pytest (unit tests for analysis logic; integration tests for API clients with recorded fixtures)

**Target Platform**: macOS/Linux developer machine (single user, no server)

**Project Type**: CLI tool / data pipeline

**Performance Goals**: Full 3-target run < 15 minutes; cached re-run < 30s/target

**Constraints**: No authentication required; no database; no cloud infrastructure; RDKit requires conda

**Scale/Scope**: 3 pre-configured targets; ~1000–5000 compounds per target; < 100MB total data on disk
