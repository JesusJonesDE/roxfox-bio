# Implementation Plan: Computational Validation Gates

**Branch**: `005-validation-gates` | **Date**: 2026-05-28 | **Spec**: [spec.md](spec.md)

## Summary

Add four computational validation gates (ADMET, MM-GBSA rescoring, selectivity docking panel, MD pose stability) as a new `pipeline validate` command. Each gate produces a pass/fail decision and markdown report. A dashboard command aggregates results across scaffolds into a wet-lab handoff decision. All gates run locally on Apple M1 Max 64 GB. The MD gate uses a 2 ns implicit solvent fast mode by default (1 hour) due to OpenMM lacking a Metal GPU backend; a 50 ns full mode is available as an explicit opt-in.

**Dependencies (existing, reused)**: `vina`, `meeko`, `rdkit`, `biopython`, `pandas`, `rich`, `httpx`, `typer`, `tenacity`

**Storage**: Files under `data/results/{gene}/` and `data/cache/{gene}/` (existing pattern). Gate results cached as JSON via existing `CacheManager`.

**Testing**: pytest (existing). Unit tests for gate logic; integration test using SCF-009 on VRK1.

**Target Platform**: macOS (Apple M1 Max, 64 GB). All computation local. No GPU acceleration for MD (OpenMM has no Metal backend).

**Performance Goals**:
- ADMET: < 60 s per scaffold
- MM-GBSA: < 10 min per scaffold (CPU)
- Selectivity panel: < 2 h per scaffold (4 × Vina docking)
- MD fast mode: < 2 h per scaffold (2 ns implicit solvent, CPU)

**Constraints**: No paid APIs. No account-required services. Offline-capable for all gates except initial structure downloads.

**Project Type**: CLI pipeline extension (follows existing `pipeline/stages/` pattern)

## Key Implementation Decisions

### Gate 1 — ADMET
- Tool: `admet-ai` (Chemprop v2 models, TDC benchmark)
- Input: SMILES string
- Output: dict of property scores
- Thresholds: BBB_Martini > 0.5, CYP1A2/2D6/3A4_Inhibitor < 0.3, Solubility logS > −4, HIA_Hou > 0.3
- No external calls at prediction time; model weights bundled with package

### Gate 2 — MM-GBSA
- Tool: `gmx_MMPBSA` (CPU, conda-forge osx-arm64)
- Forcefield: GAFF2 (small molecule) + ff19SB (protein)
- Input: receptor PDB + top pose PDBQT (converted to PDB via meeko/RDKit)
- Output: ΔG in kcal/mol
- Threshold: ΔG ≤ −7.0 kcal/mol
- GROMACS lacks Metal GPU support → CPU-only is fine for rescoring (2–5 min)

### Gate 3 — Selectivity
- Tool: existing `run_dock()` from `pipeline/stages/dock/dock.py`
- Panel: VRK2 (AF2), EGFR (1M17), CDK2 (1E9H), PLK1 (2OKR)
- Structures cached in `data/cache/shared/selectivity_panel/`
- Selectivity index = |target ΔG| / max(|off-target ΔG|); pass if SI ≥ 10
- VRK2 AlphaFold2 model downloaded from EBI API; flagged with low-confidence warning

### Gate 4 — MD
- Tool: OpenMM (CPU platform, no Metal GPU support on Apple Silicon)
- **Fast mode (default)**: 2 ns, implicit solvent (OBC/GB), ~1–2 h on M1 Max CPU
- **Full mode (--md-full)**: 50 ns, explicit solvent (TIP3P), ~1–2 days on CPU
- Forcefield: ff14SB (protein) + GAFF2 (ligand via openmmforcefields)
- PDB preparation: pdbfixer (cap termini, add missing residues/atoms)
- Analysis: MDTraj RMSD of ligand heavy atoms vs. starting pose
- Pass: mean RMSD ≤ 3.0 Å over final window (1 ns fast, 25 ns full)

### Dashboard
- Rich Table in terminal (existing Rich dependency)
- Markdown table written to `validation_dashboard.md`
- Handoff report auto-generated for any scaffold with all gates PASS

## Complexity Tracking

| Design choice | Why needed | Simpler alternative rejected because |
|---|---|---|
| Separate `gates/` subdirectory | 4 gates × different dependencies; isolation prevents import errors if one tool not installed | Flat files would mix concerns and make optional-dependency handling harder |
| Fast mode / full mode for MD | OpenMM CPU is 0.5–2 ns/day; 50 ns would take days | Removing MD gate entirely loses the highest-confidence filter; fast mode gives useful signal in 1–2 hours |
| Reuse `run_dock()` for selectivity | Selectivity is exactly docking against different targets | Reimplementing docking would duplicate 200+ lines |
